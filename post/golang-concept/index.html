Golang常见问题拾遗

Written by W1ndy with ♥ on October 20, 2020 in Golang, 2009 words

基础语法

  1. =和：=的区别？

    =是赋值

    :=是声明变量并赋值

  2. 指针的作用?

    指针类型用于传递地址，而不是传递值。因为golang的函数，所有的参数都是传递一个复制的值，如果值得体积过大，那么就会严重影响效率，而传递一个地址，就会大大提升效率。另外传递指针也能让go函数实现对变量值的修改。

  3. Go允许多个返回值吗？

    允许

  4. Go有异常类型吗？

    没有。但是有错误类型，errors实现了error的接口，只要结构体实现了error方法，就可以自定义异常。

  5. 什么是协程(Goroutine)？

    go启动协程就是使用go关键字，后面一般接一个函数或者匿名函数。

    在一个函数调用前加上go关键字，这次调用就会在一个新的goroutine中并发执行。当被调用的函数返回时，这个goroutine也自动结束。需要注意的是，如果这个函数有返回值，那么这个返回值会被丢弃。

    goroutine是在golang中执行并发任务的方式。他们仅存于go运行时的虚拟空间中，而不存在于os中。因此需要go调度器来管理他们的生命周期。

  6. 如何高效地拼接字符串？

    • +` 连接适用于短小的、常量字符串（明确的，非变量），因为编译器会给我们优化。

    • Join`是比较统一的拼接，不太灵活

    • fmt和buffer基本上不推荐

    • builder从性能和灵活性上，都是上佳的选择。

    func StringBuilder() string {
 var b strings.Builder
 b.WriteString("aaa")
 b.WriteString("bbb")
 return b.String()
}

  7. 什么是 rune 类型?

    官方回答如下：

    // rune is an alias for int32 and is equivalent to
// int32 in all ways. It is
// used, by convention, to distinguish character
// values from integer values.
   
// int32的别名，几乎在所有方面等同于int32
// 它用来区分字符值和整数值
   
type rune = int32

    rune类型实质其实就是int32，他是go语言内在处理字符串及其便捷的字符单位。我们在处理字符串的时候可以通过map[rune]int类型方便的判断字符串是否存在，其中 rune表示字符的UTF-8的编码值，int表示字符在字符串中的位置。

  8. 如何判断 map 中是否包含某个 key ？

    if _, ok := map[key]; ok {
    // 存在
}
    
if _, ok := map[key]; !ok {
    // 不存在
}

  9. Go 支持默认参数或可选参数吗？

    Go中的函数是不支持带默认值的可选参数的。

  10. defer 的执行顺序

    defer就像一个LIFO的栈，后defer的会先执行，且在函数退出时才会执行。

    defer的执行顺序在return之后，但是在返回值返回给调用方之前，所以使用defer可以达到修改返回值的目的。

  11. 如何交换 2 个变量的值？

    package main
    
import (
    "fmt"
)
    
func main() {
    var a, b = 1, 2
    fmt.Println(a, b)
    b, a = a, b
    fmt.Println(a, b)
}

  12. Go 语言 tag 的用处？

    我们可以通过Tag来增强结构体的定义，Tag会带上一些meta信息。

    Tag在运行时可以通过reflection包来读取。

    Tag最常用的大概就是在marshaling。

  13. 如何判断 2 个字符串切片（slice) 是相等的？

    首先slice相等就需要类型和长度都相同，且相等下标的值也是相等的。

    判断两个[]bytes相等？

    bytes.Equal(a, b)

    使用reflect判断slice相等?

    package main
    
import (
    "fmt"
    "reflect"
)
    
func main() {
    a := []int{1, 2, 3, 4}
    b := []int{1, 3, 2, 4}
    c := []int{1, 2, 3, 4}
    fmt.Println(reflect.DeepEqual(a, b))
    fmt.Println(reflect.DeepEqual(a, c))
}

  14. 字符串打印时，%v 和 %+v 的区别?

    1. %v 只输出所有的值

    2. %+v 先输出字段类型，再输出该字段的值

    3. %#v 先输出结构体名字值，再输出结构体(字段类型+字段的值)

  15. Go 语言中如何表示枚举值(enums)？

    使用iota模拟枚举

    const (
    Red = iota // 开始生成枚举值, 默认为0
    Blue
    Green
    yellow
)

    自定义

    type Color rune
    
const (
  Red    Color = 1
  Blue   Color = 2
  Green  Color = 3
  yellow Color = 4
)

  16. 空 struct{} 的用途?

    空对象是指没有字段的结构类型

    type Q struct{}

    空结构体的特点:

    1. 不占用内存；

    2. 地址不变

    空结构体的作用:

    用于传递信号的通道，因为不占用内存

并发编程

  1. 无缓冲的 channel 和有缓冲的 channel 的区别？

    无缓冲和有缓冲最大的区别是一个是同步的，一个是异步的。

    make(chan int) // 无缓冲，同步。只要这个数据没被接收然后所有的发送就被阻塞
make(chan int, 3) // 有缓冲，异步。无阻塞，只要通道有空间，就无需等待你的接收。

  2. 什么是协程泄露(Goroutine Leak)？

    goroutine leak，是go协程泄漏，什么是go协程泄漏，通俗来说，开启了一个goroutine，用完后，我们要正确让其结束。如果它没用了，还没结束，那就是goroutine leak。

    如何解决协程泄露?

    当主线程结束时，告知生产线程，生产线程得到通知后，进行清理工作：或退出，或做一些清理环境的工作。

  3. Go 可以限制运行时操作系统线程的数量吗？

    可以，我们可以通过两种方式对线程进行操作，创建线程池，对线程的数量进行控制。

    a) 通过for-range的方式创建有限数量的goroutine

    b) 通过channel给每个goroutine传递任务的相关信息

    package main
   
import (
 "fmt"
 "runtime"
 "sync"
 "time"
)
   
func main() {
 jobsCount := 10
 group := sync.WaitGroup{}
 var jobsChan = make(chan int, 3)
 poolCount := 3
 for i := 0; i < poolCount; i++ {
     go func() {
         for j := range jobsChan {
             fmt.Printf("hello %d\n", j)
             time.Sleep(time.Second)
             group.Done()
         }
     }()
 }
 for i := 0; i < jobsCount; i++ {
     jobsChan <- i
     group.Add(1)
     fmt.Printf("index: %d,goroutine Num: %d\n", i, runtime.NumGoroutine())
 }
 group.Wait()
 fmt.Printf("Done!")
}

    也可以使用环境变量 GOMAXPROCS 或 runtime.GOMAXPROCS(num int) 设置。GOMAXPROCS 的默认值等于 CPU 的逻辑核数，同一时间，一个核只能绑定一个线程，然后运行被调度的协程。对于 CPU 密集型的任务，若该值过大，例如设置为 CPU 逻辑核数的 2 倍，会增加线程切换的开销，降低性能。对于 I/O 密集型应用，适当地调大该值，可以提高 I/O 吞吐率。
